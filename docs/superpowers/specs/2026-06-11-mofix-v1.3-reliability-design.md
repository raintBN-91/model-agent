# MoFix v1.3 — 可靠性演进设计文档

## 概述

在 MoFix v1.2 基础上，以**零改动原代码**为前提，在 `F:\7-workspace\mofix-new\mofix-v1.3` 下建立新版。聚焦两个方向：**A. 测试体系优先** + **C. 增量加固**。

## 设计原则

- **不修改 v1.2 任何现有代码** — 全部变更发生在 v1.3 目录
- **测试先行** — 先建测试基础设施，再补核心模块覆盖
- **增量不改架构** — 加固层只加防护不重构
- **离线可运行** — P0 测试不依赖任何外部服务

### CI 配置

```yaml
# .github/workflows/ci.yml
name: CI
on: [push, pull_request]
jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with: { python-version: "3.11" }
      - run: pip install ruff
      - run: ruff check app/ tests/
  unit-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with: { python-version: "3.11" }
      - run: pip install -r requirements.txt
      - run: pip install pytest pytest-asyncio pytest-cov pytest-mock
      - run: pytest tests/ -m "not slow and not integration" --cov=app --cov-report=term-missing
  integration-test:
    runs-on: ubuntu-latest
    if: github.event_name == 'push' && github.ref == 'refs/heads/main'
    steps:
      - uses: actions/checkout@v4
      - ...
      - run: pytest tests/ -m integration
        env:
          OPENAI_API_KEY: ${{ secrets.OPENAI_API_KEY }}
```

## 增量加固（方案 C）

### C1. 外部调用防护

| 位置 | 问题 | 加固措施 |
|------|------|---------|
| `app/tools/doc_tools.py` | 超时值散落（600s, 1800s） | 提取为模块常量 `_SUBPROCESS_TIMEOUT`、`_ONE_CLICK_TIMEOUT` |
| `app/tools/claude_tools.py` | `_invoke_claude_skill_async` 无整体超时 | 增加 `asyncio.wait_for()` 兜底（默认 300s） |
| `app/tools/workflow/mcp_integration.py:415` | `_discover_tools` 缺超时 | 给 `send_and_receive` 增加 timeout 参数 |
| `app/core/llm_factory.py` | LLM 调用无重试 | 新增 `retry_llm_call()` 装饰器（指数退避 3 次） |
| `app/tools/workflow/planner.py:448` | `_call_llm` 失败后仅降级 | 先重试再降级 |

### C2. 输入校验加强

| 模型/位置 | 加固内容 |
|-----------|---------|
| `ChatCompletionRequest.max_tokens` | 范围 [1, 131072] |
| `ChatCompletionRequest.temperature` | 范围 [0, 2] |
| `ChatCompletionRequest.c_id` | 最大长度 128 |
| `ChatMessage.content` | 空字符串转为 None 的统一处理 |
| `ChatCompletionChunk.created` | 验证时间戳合理性（非未来 >1h） |

### C3. 结构化日志补全

| 位置 | 补充日志 |
|------|---------|
| `app/core/agent_engine.py` | build_agent 成功/失败、tools 加载数量 |
| `app/tools/workflow/planner.py:213` | plan() 开始/完成/失败 + LLM token 消耗 |
| `app/tools/workflow/executor.py` | 每步开始、结束、重试、checkpoint 写入 |
| `app/services/chat_service.py:87` | 命令执行入口日志（含 trace_id） |
| `app/api/chat.py:16` | 请求入口日志，注入 trace_id 传播链 |

### C4. 错误响应统一

| 位置 | 当前行为 | 加固后 |
|------|---------|--------|
| `chat_service.py` 命令执行失败 | `f"[错误] 执行失败：{e}"` | 统一 JSON 错误格式 |
| `handle_api_error` | 抛出异常 | 确保所有 LLM 调用点在外层 try/except |
| 骨架工具返回值 | 占位字符串 | 调整为 `{"status": "not_implemented", "message": "..."}` 格式 |
