# 中文单一效果验证提示词

本文件用于“单一效果验证”。每次只验证一个能力点，避免一次请求混合多个目标。

## 适用条件

- 用户只要求检查一个能力点
- 用户只要求输出一个局部方案、模板、命令清单或最小修改点
- 任务不要求完成完整训练、导出、推理、交付闭环

若当前需求超出单点验证范围，应切换到整体适配流程，而不是继续使用本文件。

## 使用方式

- 在提示词前显式写：`使用 $cv-ascend-adapt`。
- 只选择一个测试用例。
- 一次只验证一个目标能力，不混合多个动作。
- 验收口径固定为：动作、证据、结论、失败修复建议。
- 若检查项涉及真实 NPU 设备，请显式说明是否需要绕过当前沙箱执行。
- 若当前环境不可执行真实 NPU 命令，必须明确说明“仅完成脚本生成，待客户在真实昇腾机器执行”。
- 单一效果验证默认不生成整套交付文档，除非用例本身就是“仅验证交付报告结构”。

## 用例1：仅验证环境收敛输出

提示词：
```text
使用 $cv-ascend-adapt，仅做“环境收敛检查”，输出环境矩阵模板和需要采集的命令清单，不做任何代码修改；环境矩阵必须包含 Python、PyTorch、torchvision、torch_npu、CANN、驱动、固件，并检查 torchvision 与 torch 的兼容关系、torch_npu 与 torch 的版本匹配关系；若 npu-smi 或 torch.npu 探测受沙箱影响，可绕过沙箱执行。
```

通过标准：
- 明确版本矩阵字段（Python/PyTorch/torchvision/torch_npu/CANN/驱动/固件）
- 明确 torchvision 与 torch 的兼容检查
- 明确 torch_npu 与 torch 的版本匹配检查
- 给出可执行的采集命令
- 不包含训练改造动作
