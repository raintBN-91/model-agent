# 阶段1：环境收敛

## 输入信息

- 阶段0的结构化需求分析结果
- 基线环境记录
- 当前执行环境信息
- 项目依赖线索和运行入口线索

## 必做动作

- 固化 Python、PyTorch、torchvision、torch_npu、CANN、驱动、固件版本
- 环境矩阵至少包含：
  - Python
  - PyTorch
  - torchvision
  - torch_npu
  - CANN
  - 驱动
  - 固件
- 校验 `torchvision` 与 `torch` 的版本兼容关系
- 校验 `torch_npu` 与 `torch` 的主次补丁版本匹配关系
- 使用 `npu-smi info` 校验设备可见性
- 补充 `torch.npu.is_available()` 与 `torch.npu.device_count()` 的运行时探测
- 校验容器挂载、设备节点与执行上下文
- 记录容器启动命令、必要挂载与设备节点
- 记录可复现实验命令
- 先判断当前环境是否具备真实 NPU 执行条件
- 当当前执行环境存在额外沙箱、会重挂 `/dev` 或屏蔽 NPU 设备节点时，NPU 相关检查切换到真实容器或宿主机视角执行，必要时绕过当前沙箱
- 若 `npu-smi info`、`torch.npu.is_available()` 与业务方本地结果冲突，优先排查执行上下文是否被沙箱隔离

## 禁止动作

- 未检查版本兼容关系就判定环境通过
- 未判断真实 NPU 执行条件就进入真实训练或真实推理
- 在沙箱导致 `/dev` 视图异常时，直接判定容器环境不通过
- 在真实 NPU 不可执行时输出“已完成真实训练验证”结论

## 完成标准

- 运行时矩阵已完整固化
- 版本兼容与匹配关系已核验
- NPU 可见性与运行时探测结果已记录
- 容器挂载、设备节点、执行上下文已记录
- 已明确当前环境是“可真实执行”还是“不可真实执行”

## 失败处理

- 若当前环境具备真实执行条件，继续后续真实训练与验证
- 若当前环境不具备真实执行条件：
  - 停止真实训练动作
  - 转为生成脚本和执行说明
  - 要求客户将脚本复制到昇腾机器执行

## 下一阶段

- 真实执行条件判定完成后，进入阶段1.5

## 输出结果

- `env_matrix.md` 所需环境矩阵内容
- 版本兼容检查记录
- NPU 运行时探测记录
- 容器与挂载信息记录
- 真实执行条件判定结论
- 可复现实验命令清单
