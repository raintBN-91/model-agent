# 数据错误分级处置

## P0 严重

- 定义：
  - 阻断数据集构建或 batch 生成。
- 典型现象：
  - `KeyError: dataset_type`
  - `build_mm_dataset` 构建失败
  - collate 崩溃且无法产出 batch
- 立即动作：
  - 停止验证流程。
  - 优先修复注册与数据契约。
  - 重新执行最小数据构建检查。

## P1 高

- 定义：
  - 数据可构建，但样本语义或兼容性错误。
- 典型现象：
  - `__getitem__` 缺少必需字段
  - 路径类型不匹配（`str` vs `list[str]`）
  - 源预处理逻辑部分失效
- 动作：
  - 修补字段映射与路径标准化。
  - 重新执行数据验收清单。

## P2 中

- 定义：
  - 不阻断运行但会提升后续失败风险的问题。
- 典型现象：
  - 错误信息可读性弱
  - 非复用改动缺少适配说明
- 动作：
  - 在最终报告前补强诊断与文档说明。
