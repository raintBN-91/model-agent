# 分阶段交付门禁

## P0 发现阶段

- [ ] 产出 `source_target_inventory.yaml`
- [ ] 产出 `entrypoint_checklist.md`
- [ ] 源仓入口可追溯到具体文件

## P1 映射阶段

- [ ] 产出 `compatibility_matrix.yaml`
- [ ] 产出 `reuse_vs_adapt.md`
- [ ] 每项“必须适配”均有原因

## P2 实施阶段（Model/Data/Config）

- [ ] 三个子技能各自产出验收清单
- [ ] 产出 `changeset_manifest.yaml`
- [ ] 改动路径未越界

## P3 验证阶段

- [ ] 产出 `verification_report.md`
- [ ] 产出 `evidence.json`
- [ ] 失败时产出 `failed_cases.md`

## P4 汇总阶段

- [ ] 产出 `migration_report.md`
- [ ] 产出 `risk_register.yaml`
- [ ] 产出 `next_actions.yaml`

## 阶段推进规则

- [ ] 当前阶段交付不完整，禁止进入下一阶段
- [ ] 任一阶段失败，必须回流到责任技能修复
