# 阶段0：需求分析与信息提取

## 输入信息

- 用户提供的全部输入信息，优先级最高，包括需求描述、补充说明、附件说明、路径、命令、报错、截图说明与历史上下文
- 用户需求文本
- 模型源码或项目路径
- 训练框架信息（若已提供）
- 数据集线索、预训练权重路径、参考资料链接
- 用户明确要求的适配范围：单卡训练、多卡训练、推理、导出
- 用户提供的网页链接、Issue 链接、README 链接、论文链接或文档链接

## 必做动作

- 先对用户提供的原始输入信息做完整分析，不得跳过用户已经给出的上下文
- 解析并提取关键信息：
  - 模型类型与架构
  - 任务类型（分类、检测、分割、检索等）
  - 框架信息（MMPretrain、MMDetection、原生 PyTorch 等）
  - 数据集信息（名称、路径、格式）
  - 预训练模型路径
  - 适配范围（单卡、多卡、推理、导出）
  - 参考资料链接
- 若用户提供网页链接，必须先获取链接内容，再基于实际内容进行分析与信息提取，不得仅根据链接文字、URL 片段或主观猜测下结论
- 对网页链接内容至少提取以下信息（按实际内容适用）：
  - 模型或项目定位
  - 安装与运行要求
  - 训练、推理、导出相关入口
  - 数据集或权重获取方式
  - 与适配任务直接相关的约束、命令或注意事项
- 验证信息完整性：
  - 模型类型和任务类型必须明确
  - 框架信息或源码路径必须明确
  - 若涉及训练，数据集信息必须明确或提供获取方式
- 生成结构化需求分析报告（JSON 或 Markdown）
- 根据需求信息确定后续执行路径
- 固定源码提交号与基线指标
- 保存基线训练命令与至少一个推理样例
- 记录基线可用环境

## 禁止动作

- 忽略用户已经提供的路径、链接、命令、报错或补充说明，转而凭空假设任务背景
- 对网页链接未读取正文内容就直接做分析、提取信息或输出结论
- 在模型类型、任务类型、框架信息均不明确时进入后续阶段
- 在训练任务缺少数据集信息或获取方式时直接推进真实训练
- 未形成结构化分析结果就开始改造
- 未固化基线提交号、基线命令、基线样例就进入环境和训练阶段

## 完成标准

- 用户提供的输入信息已完成归类、消歧和提取
- 用户提供的网页链接已完成内容获取，并已提取出与任务相关的有效信息
- 需求中的关键字段已提取完成
- 缺失信息已明确标记
- 结构化需求分析报告已生成
- 基线提交号、基线指标、基线命令、基线推理样例、基线环境已记录
- 后续执行路径已经明确

## 失败处理

- 若关键信息缺失，暂停并要求用户补充
- 若网页链接无法访问或无法获取正文内容，明确记录失败原因、缺失影响与待补充项，再决定是否暂停阶段0
- 若源码路径或框架信息缺失，暂停环境与训练链路推进
- 若基线无法固化，明确记录阻塞原因，并停止进入阶段1

## 下一阶段

- 信息完整且基线已固化时，进入阶段1

## 输出结果

- 结构化需求分析报告
- 用户输入信息提取结果
- 网页链接内容提取结果（若用户提供链接）
- 基线信息记录：
  - 源码提交号
  - 基线指标
  - 基线训练命令
  - 基线推理样例
  - 基线可用环境
