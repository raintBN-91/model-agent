# 阶段3：分布式与精度兼容

## 输入信息

- 阶段2已打通的训练链路
- 分布式启动脚本与通信后端配置
- AMP、autocast、collective、同步逻辑相关实现

## 必做动作

- NPU 路径优先使用 HCCL
- 在进程组初始化前绑定本地设备
- 为分布式同步逻辑补充“已初始化”保护
- 为仅分布式场景使用的同步逻辑补充保护
- 修复 collective 路径 dtype 兼容问题，避免 HCCL 路径出现 `float64`
- 修复 autocast `device_type` 兼容问题
- 第三方库 CUDA 硬编码改造必须隔离处理

## 禁止动作

- 在进程组初始化前不绑定设备
- 放任仅分布式场景的同步逻辑在单卡路径误触发
- 保留 HCCL 路径中的 `float64` collective
- 将第三方库 CUDA 硬编码改造与主流程改造混杂处理

## 完成标准

- 分布式路径使用 HCCL 且设备绑定顺序正确
- 同步逻辑具备初始化保护
- dtype 与 autocast 兼容问题已修复
- 不存在未解决的 HCCL、dtype 或 autocast 错误

## 失败处理

- 若多卡初始化失败，优先排查设备绑定顺序和进程组初始化时机
- 若出现 HCCL/dtype/autocast 错误，停止进入导出阶段，先完成兼容修复
- 若问题来自第三方库 CUDA 硬编码，隔离修复并单独记录

- 分布式与精度稳定后，进入阶段4

## 输出结果

- 分布式兼容改造记录
- 精度兼容修复记录
- 多卡与 AMP 验证结果
