# 阶段5：报告与交付

## 输入信息

- 阶段0到阶段4的全部证据、日志、命令、指标和产物
- 文档骨架
- 训练、导出、推理的最终结论

## 必做动作

- 使用标准文档模板输出结果
- 文档骨架由 `scripts/scaffold_docs.py` 生成，不手写空模板
- 完成以下交付物：
  - `env_matrix.md`
  - `adaptation_checklist.md`
  - `issue_matrix.md`
  - `adaptation_report.md`
- 每个结论必须附命令、日志或指标证据
- 若完成真实训练验证，还应固化：
  - 训练命令
  - work_dir
  - 日志路径
  - loss 曲线或标量文件路径
  - 关键 loss 点
  - checkpoint 路径
  - 最终验证指标
  - 收敛结论
  - 通用环境信息：操作系统、Python、PyTorch、torchvision、torch_npu、CANN、驱动、固件、npu-smi、NPU 可见性
  - 项目/框架信息
  - 推理性能统计结果
  - 推理精度统计结果
  - 数据集路径与标注格式说明（含字段示例）
  - 单卡与多卡可执行命令（真实路径）
  - 导出后文件清单（meta / onnx / om / log）
  - 依赖安装步骤（标准安装 + 失败回退 + 验证）
  - 推理去重策略及去重前后统计
  - 若项目存在框架层信息，额外固化项目名称、训练框架、框架版本、训练入口脚本、分布式启动脚本
  - 若训练为多卡 AMP，额外标记卡数、分布式后端、是否启用 AMP、是否出现 loss scale 调整、相关 warning 是否阻塞
- 真实训练证据固化格式应覆盖：
  - 完整可复现训练命令
  - 数据集信息
  - 关键 loss 节点
  - 前期与后期 loss 对比
  - 最终指标
  - 是否收敛、是否通过、是否存在阻塞问题
  - 风险说明
  - 若真实训练依赖绕过沙箱执行，必须显式注明
- 若当前环境不可执行真实 NPU，应固化：
  - 训练脚本
  - 启动脚本
  - 客户侧执行命令
  - 预期日志
  - 验收说明
  - 明确标记“待客户在真实昇腾机器执行”
- 清单与问题矩阵未完成时不得标记完成

## 禁止动作

- 未生成标准交付文档就宣称任务完成
- 结论没有命令、日志或指标证据支撑
- 清单与问题矩阵未完成时标记完成
- 在当前环境无法真实执行时输出真实验证已完成结论

## 完成标准

- `env_matrix.md`、`adaptation_checklist.md`、`issue_matrix.md`、`adaptation_report.md` 全部完成
- 报告中的每个结论均有证据支撑
- 若存在真实训练，训练证据完整固化
- 若不存在真实训练条件，客户侧执行交付完整固化
- 证据区、风险区、结论区完整

## 失败处理

- 若证据不足，回到对应阶段补齐命令、日志或指标
- 若训练证据缺失，回补命令、日志、loss 节点、checkpoint、最终指标
- 若导出或推理证据缺失，回补性能、精度、产物与报告
- 若仅完成脚本生成，明确标记待客户执行，不得误报完成

## 验收清单

- 训练：
  - 单卡启动任务通过
  - 真实图片 1 epoch 任务通过
  - 若用户要求收敛验证，长周期训练通过
- 稳定性：不存在未解决的 HCCL、dtype 或 autocast 错误
- 导出：同一输入可稳定复现 OM 构建结果
- 推理：输出格式与阈值行为已验证
- 证据：
  - 有训练日志
  - 有关键 loss 点
  - 有首轮与末轮 loss 对比
  - 有前 10 轮与后 10 轮均值对比
  - 有 loss 曲线或等价标量文件
  - 有最终验证指标
  - 有 checkpoint 路径
  - 有明确的“收敛/未收敛”结论
- 报告：证据区与风险区已完成
