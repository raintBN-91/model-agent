## 问题描述

### Motivation.

1.Deepseek R1 has implemented W4A8 per-group (group_size 256) quantization. W4A8 per-channel is realized by further reducing the quantization granularity. Deepseek R1 has implemented W4A8 per-group (group_size 256) quantization. W4A8 per-channel is implemented by further reducing the quantization granularity, which further improves performance.
2.Qwen3-235B/30B is quantized only by W8A8. This requirement further reduces the number of bits by W4A8 per-channel to improve performance.

### Proposed Change.

**Proposed Change.**

- The is_per_channel_weight attribute is added. The value 0 of group_size indicates whether the weight is per-channel. To be compatible with the weight format that has been adapted before, the logic that requires special processing uses this attribute to determine the weight format. Different code branches are used to reuse the original code.
- Parameters generated by per-group quantization, such as weight_scale_second, are no longer created or read by default, and the scale calculation logic does not need to multiply scale and scale_second.
- Verify the GMM operator. Currently, the W4A8 per-channel scenario is supported. Only the shape of the scale changes. In the per-channel scenario, the value is (e, 1, n).
- The GMM w4a8 supports NZ, and the weight processing phase converts the format to NZ.
