# 模型错误分级处置

## P0 严重

- 定义：
  - 立即阻断模型构建或 forward 契约。
- 典型现象：
  - `KeyError: model_id`
  - 必需模型导入出现 `No module named ...`
  - 模型输出缺少 `.loss`
- 立即动作：
  - 停止后续任务。
  - 优先修复注册或加载路径。
  - 重新执行最小模型构建检查。

## P1 高

- 定义：
  - 模型可构建，但训练语义可能错误。
- 典型现象：
  - token 注入不匹配
  - embedding resize/init 不匹配
  - `from_pretrained` 签名处理冲突
- 动作：
  - 应用定向补丁。
  - 重新执行模型验收清单。

## P2 中

- 定义：
  - 不阻断运行但影响可维护性的风险。
- 典型现象：
  - 适配理由不完整
  - 路径/风格轻微不一致但未导致运行中断
- 动作：
  - 在最终汇总前修复。
  - 持续更新证据与理由说明。
