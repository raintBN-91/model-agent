## Step 2 检查算子现有 UT 框架

### Step 2.1 查看算子目录结构

```bash
# 确认算子目录位置
find ${repo_path} -maxdepth 2 -type d -name ${op_name}

# 查看算子目录结构
tree ${op_path}
```

根据算子 `ut` 目录下的文件情况，判断算子是否已有 UT 框架

### Step 2.2 编译运行现有用例

**⚠️ 本步骤不能跳过！即使用户要求"重构 UT 框架"，也必须先验证旧框架是否正常。**

如果算子已有 UT 框架，编译运行 `${test_model}` 涉及的用例，确认：
- [ ] 已有用例可以编译通过
- [ ] 已有用例全部 PASS
- [ ] 可以获取到覆盖率数据

**如果现有用例无法运行通过，或无法获取覆盖率，使用 `question` 工具向用户询问：**
1. 尝试自动修复现有用例
2. 立即结束会话，由用户确认编译环境是否正常
