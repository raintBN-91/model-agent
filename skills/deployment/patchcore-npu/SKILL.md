# SKILL.md — PatchCore 异常检测 昇腾 NPU 性能调优

## 基本信息

| 项目 | 内容 |
|------|------|
| 赛道 | 赛道二：性能调优 (800I A2) |
| 模型 | PatchCore (WideResNet50) |
| 框架 | PyTorch + timm |
| CANN 版本 | 8.5.1 |
| torch 版本 | 2.9.0 |
| NPU 型号 | Ascend 910 (双芯片) |
| 精度模式 | BF16 + HF32 |

## 优化策略

### 1. BF16 混合精度
- 模型和输入张量转为 torch.bfloat16
- 延迟降低 23%（6.5ms → 5.0ms）
- 显存降低 13%（0.30 GB → 0.26 GB）

### 2. HF32 算力加速
- 开启 TORCH_NPU_ALLOW_HF32=1
- 利用 Ascend 910 HF32 算力，矩阵乘法速度提升
- 与 BF16 叠加，吞吐量提升 28%（154.6 → 198.4 FPS）

### 3. 推理预热
- 首次推理前执行一次预热，消除冷启动偏差
- torch.npu.synchronize() 确保准确计时

## 性能结果

| 指标 | 优化前 (FP32) | 优化后 (BF16+HF32) | 提升 |
|------|-------------|-------------------|------|
| 平均延迟 | 6.5 ms | 5.0 ms | 23% |
| P95 延迟 | 6.6 ms | 5.2 ms | 21% |
| 吞吐量 | 154.6 FPS | 198.4 FPS | 28% |
| 峰值显存 | 0.30 GB | 0.26 GB | 13% |

## 调优总结

PatchCore (WideResNet50) 成功在昇腾 Ascend 910 NPU 上完成推理部署。
通过 BF16 混合精度 + HF32 算力加速，吞吐量提升 28%（154.6 → 198.4 FPS），
延迟降低至 5.0ms，显存仅 0.26 GB。纯 PyTorch 实现，无自定义算子依赖。
