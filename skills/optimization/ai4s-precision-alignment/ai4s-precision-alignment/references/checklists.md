# 核对清单

## 通用前置清单

- 已先检查官方 README、官方示例、benchmark 或官方文档是否给出基线
- 已记录基线来源优先级
- 尚未确认正式基线前，没有先写或运行精度对比脚本
- 没有因为本地 CPU 路径更容易跑通，就默认改成 CPU vs NPU
- 如果做了 CPU 对比，已明确它是否只是调试用途
- 如果阈值会影响结论，阈值已在比较前声明

## 训练精度对齐清单

- 官方训练基线是否存在，已确认
- 若官方训练基线不存在，已向用户确认 GPU 基线提供方式
- 若由用户提供 GPU 基线，目录路径和文件含义已确认
- 若用户提供的是目录，已先运行 `scripts/scan_baseline_bundle.py`
- 两侧代码版本一致
- 使用同一个 checkpoint 或同样的初始化方式
- 数据集内容和顺序一致
- seed 和随机性控制一致
- batch size 与梯度累积一致
- 优化器、scheduler 和 warmup 一致
- loss 函数和 reduction 模式一致
- 精度模式与 autocast 边界一致
- 分布式 world size 和 reduction 语义一致
- 日志打印频率和指标聚合方式一致

## 推理精度对齐清单

- 官方推理基线是否存在，已确认
- 如果官方已给基线，已优先直接比较官方值与 NPU 结果
- 若官方推理基线不存在，已向用户确认 GPU 基线提供方式
- 若由用户提供 GPU 基线，目录路径和文件含义已确认
- 若用户提供的是目录，已先运行 `scripts/scan_baseline_bundle.py`
- 没有把本地 CPU 输出默认当作正式基线
- 如果做了 CPU 对比，已明确写成“调试辅助”而不是“最终对齐结论”
- checkpoint 和权重转换路径一致
- tokenizer 或特征提取器一致
- 归一化、resize、crop、padding 规则一致
- prompt 模板和 special token 一致
- 解码或阈值参数一致
- batch shape 与动态 shape 行为一致
- 指标脚本和后处理一致
- 样本集及顺序一致

## Dump 采集清单

- 正式基线来源已确认
- 基线目录或 dump manifest 已确认
- 已保存最小可复现输入
- 尽量固定为单卡、单进程
- seed 与动态 shape 设置已固定
- dump 范围已缩小到特定 block、layer 或 token range
- 已采集输入 tensor、关键中间 tensor 和最终输出
- 已记录 dtype、shape、scale、device 和 layout
- 已记录工具版本、驱动版本、固件版本和框架版本
- 已记录输出目录和命名规范
- 已写 manifest，确保基线与昇腾 dump 能一一配对

## 结论输出清单

- 明确说明是否对齐
- 明确说明基线来源是什么
- 明确说明比较覆盖范围是什么
- 明确说明使用了什么阈值或预期
- 明确说明阈值是在比较前确定的
- 明确说明最关键证据是什么
- 明确说明最可能的根因是什么
- 明确说明下一步具体动作是什么
- 如果只完成样例级推理对齐，已避免把结论写大
- 如果只做了 CPU 调试对比，已避免把它写成正式精度对齐结论
