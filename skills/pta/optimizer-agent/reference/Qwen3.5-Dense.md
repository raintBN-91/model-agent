# Qwen3.5 Dense 架构模型优化参考

## 模型特征

Qwen3.5 Dense 系列（0.8B ~ 32B）在 Ascend NPU 上的优化具有共性规律：

- **Dense 架构**：无 MoE，所有参数参与推理，显存占用与参数规模线性相关
- **Prefill 密集**：长上下文场景下，Prefill 阶段占推理时间 60-80%
- **对图编译敏感**：Dense 模型的计算图规则，图编译收益稳定

## 优化优先级建议

| 优先级 | 优化项 | 适用场景 | 预期收益范围 |
|--------|--------|---------|-------------|
| P0 | Prefix Caching | 共享前缀（多轮对话/系统提示词） | +20~70% |
| P0 | 图编译 | 通用（去掉 --enforce-eager） | +20~60% |
| P0 | 异步调度 | 高并发场景 | 调度耗时 -15~20% |
| P1 | Max Num Batched Tokens | 显存充足时增大 | +5~15% |
| P1 | Block Size | 根据模型和负载调整 | 模型相关 |
| P2 | Chunked Prefill | 长输入序列 | 改善 TTFT |
| P2 | Attention Mask 压缩 | 序列 >8k | 长序列优化 |

## 协同效应

Prefix Caching 和 图编译 组合使用时收益通常大于单独叠加，原因：
- 图编译减少算子调度开销
- Prefix Caching 复用 KV Cache 计算
- 两者作用于不同瓶颈，收益可叠加

## 参考数据

基于 Qwen3.5 Dense 系列实测：
- **小模型（0.8B）**：单卡 910B，吞吐 1.17→2.01 req/s（+72%），时延 50.5→29.7s（-41%）
- **中等模型（7B~14B）**：单卡或 TP2，图编译和 Prefix Caching 仍为核心收益点
- **大模型（32B）**：需 TP 多卡，异步调度和 FlashComm 成为额外收益点
