# 配置错误分级处置

## P0 严重

- 定义：
  - 阻断配置解析或初始化。
- 典型现象：
  - strict schema 字段触发 `unexpected keyword argument`
  - 模型或数据 plugin 关联缺失
  - `model_id` 或 `dataset_type` 解析无效
- 立即动作：
  - 停止流水线。
  - 优先修复 strict/extra 分层与注册关联。
  - 重新校验配置解析链路。

## P1 高

- 定义：
  - 配置可解析，但会导致运行行为不稳定或错误。
- 典型现象：
  - `apply_modules` 模式无效
  - 目标模态所需模型/数据参数缺失
- 动作：
  - 修正映射与模块模式。
  - 重新执行配置验收清单。

## P2 中

- 定义：
  - 不会立刻崩溃但影响质量的配置问题。
- 典型现象：
  - 映射决策可追溯性差
  - 可选参数缺少文档说明
- 动作：
  - 在最终汇总前补充映射说明与决策理由。
